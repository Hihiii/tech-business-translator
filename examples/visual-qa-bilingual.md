# Example: Bilingual Visual QA

## Prompt

Compare the implemented dashboard with the provided design screenshot. Report mismatches in English and Traditional Chinese.

## Expected Traits

- Route to visual QA report.
- Require source design, implementation screenshot, viewports, and severity.
- Avoid claiming exact match without rendered evidence.

## English Sample

Visual QA should compare the rendered implementation against the source screenshot at the specified desktop and mobile viewports. Findings should be grouped by severity, with evidence and recommended code areas for each mismatch.

## Traditional Chinese Sample

視覺 QA 應在指定的桌面與手機視窗下，將已渲染的實作畫面與原始設計截圖比對。發現問題應依嚴重度分組，並為每個差異提供證據與可能需要調整的程式碼區域。
