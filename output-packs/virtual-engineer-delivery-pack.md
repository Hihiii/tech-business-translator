# Virtual Engineer Delivery Pack

Use when a broad engineering or research task requires planning, execution, QA, and human-reviewable delivery.

## Required Inputs

- User goal.
- Scope and constraints.
- Source materials or codebase.
- Target audience.
- Required language.
- Required deliverables.
- Validation requirements.

## Outputs

1. Project plan.
2. Delegation or workstream plan.
3. Execution summary.
4. Validation report.
5. Risk and assumption log.
6. Final stakeholder-ready deliverable.

## Quality Rules

- Every workstream must have a validation method.
- Final output must identify open risks and assumptions.
- Research claims require citations or source references.
- Visual claims require screenshots or explicit inspection notes when tools are available.
- Code changes require tests or a clear explanation for why tests could not run.
