# Security Communication Pack

Use when a security issue needs coordinated executive, legal, customer, and support communication.

## Required Inputs

- Security issue or advisory ID.
- Affected versions, systems, or data categories.
- Evidence of exploitation or exposure.
- Remediation status.
- Customer action required.
- Legal review status.

## Outputs

1. Executive security brief.
2. Legal-safe customer advisory.
3. Customer FAQ.
4. Internal support talking points.
5. Remediation action tracker.

## Quality Rules

- Do not say "breach" unless confirmed.
- Use "current evidence" when facts may change.
- Include customer action required.
- Keep technical appendix separate from customer-facing summary.
- Preserve bilingual consistency if output is bilingual.
