# Stakeholder Incident Communication Pack

Use when a single technical incident needs multiple audience-specific outputs.

## Required Inputs

- Incident summary.
- Affected users or systems.
- Timeline.
- Current status.
- Known impact.
- Unknowns.
- Customer action required.

## Outputs

1. Executive summary.
2. Internal Slack update.
3. Client email.
4. Customer FAQ.
5. Support talking points.
6. Post-incident action tracker.

## Quality Rules

- Keep facts consistent across all outputs.
- Use the same numbers and time windows.
- Make client-facing versions legally safe.
- Keep internal-only details out of customer outputs.
- Provide bilingual output if requested.
